# ADR 004 - Product Documentation As Source Of Truth

## Contexto

El alcance funcional del marketplace es amplio y puede desordenarse rapido si cada iteracion inventa supuestos.

## Decision

Usar documentacion de producto y arquitectura como fuente principal de contexto antes de implementar.

## Ventajas

- reduce contradicciones entre agentes y fases;
- mejora la continuidad del trabajo;
- hace visible el alcance real y lo que queda fuera.

## Desventajas

- exige mantenimiento disciplinado;
- puede quedar obsoleta si no se actualiza junto con el codigo.

## Consecuencias

`AGENTS.md` obliga a leer los documentos clave y las decisiones importantes deben registrarse como ADR.
